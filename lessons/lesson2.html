Understanding Authorization

Authorization determines what an authenticated user is allowed to do. While authentication asks "Who are you?", authorization asks "What are you allowed to access?"

Access Control Models

  • Discretionary Access Control (DAC): Resource owners control access permissions
  • Mandatory Access Control (MAC): System-enforced access based on security labels
  • Role-Based Access Control (RBAC): Permissions assigned through roles
  • Attribute-Based Access Control (ABAC): Dynamic access based on attributes

Role-Based Access Control (RBAC)

RBAC is the most common access control model in enterprise environments:

  • Users: Individual people or entities
  • Roles: Job functions or responsibilities
  • Permissions: Specific actions or operations
  • Sessions: Mapping of users to roles

Principle of Least Privilege

Users should only have the minimum access necessary to perform their job functions. This principle helps minimize the potential damage from compromised accounts.

Access Control Implementation

  • Define clear roles and responsibilities
  • Implement role hierarchies when appropriate
  • Regular access reviews and audits
  • Automated provisioning and deprovisioning
  • Separation of duties for critical operations

Common Authorization Vulnerabilities

  • Privilege escalation attacks
  • Insecure direct object references
  • Missing function-level access controls
  • Inadequate session management
  • Role mining and inference attacks
